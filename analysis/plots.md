# Plot Reference

Documentation for all figures generated by `make_plots.py`.
Output root: `analysis/out/k5_results/`.

## File naming conventions

- `c2` - 2 MCMC chains (the primary arm; all comparisons use this)
- `kt1 / kt2 / kt3 / kt5` - number of true mixture components in the data-generating process
- `by_ktrue` - a single figure covering all four k_true values at once
- Samplers: `bayesm` (Gibbs/RW-Metropolis, R), `nuts` (No-U-Turn Sampler), `hmc` (Hamiltonian MC)

---

## Delta (Delta) - population-level heterogeneity

Delta is the D x P = 2 x 4 matrix linking demographic covariates to preference parameters.
Each element Delta_{d,p} describes how demographic z_d shifts the mean utility weight for
parameter p across the population. Recovering Delta well means the model correctly attributes
population-level heterogeneity to the right covariates.

Element labels follow the convention `Delta_{d,p} (z_d:param)`:
- rows d = 1 (z1), 2 (z2) - the two demographic variables
- cols p = 1..4 = Alt1, Alt2, Alt3, Price

### delta/bias/plots/ - Bias of Delta elements

**Files:** `delta_bias_elements_c2_kt{k}.png` (one per k_true)

**What is shown:** For each of the 8 Delta elements, a boxplot per sampler of the signed
estimation error `bias = post_mean - true_value`. Each point is one replicate seed (one
simulated dataset); the box summarizes variability across 100 seeds.

**Y-axis:** Free scale per panel. Each element has its own axis so that a large range in one
element does not compress the others.

**Reference line:** None (zero bias means the dashed horizontal would sit at y=0; not drawn
to keep the panels clean - look for boxes centered on zero).

**How to read it:**
- Box centered near 0 - sampler recovers this element without systematic over/underestimation.
- Box shifted above 0 - sampler consistently overestimates this element.
- Box shifted below 0 - sampler consistently underestimates.
- Wide box / tall jitter cloud - high variability across datasets; this element is hard to pin
  down regardless of sampler.

**Good result:** All three samplers showing boxes centered near zero, with similar spread.
Differences in spread indicate one sampler is more efficient at recovering this element.

---

### delta/sd/plots/ - Posterior standard deviation of Delta elements

**Files:** `delta_sd_elements_c2_kt{k}.png` (one per k_true)

**What is shown:** For each Delta element, a boxplot per sampler of the posterior standard
deviation `post_std` reported by the sampler. Each point is one seed.

**Y-axis:** Free scale per panel.

**How to read it:**
- Low post_std - the sampler is confident about this element (tight posterior).
- High post_std - the sampler is uncertain (wide posterior).
- Differences between samplers in post_std, for the same element and k_true, indicate that one
  sampler places tighter credible intervals. This is informative alongside coverage: a sampler
  with low SD and good coverage is well-calibrated; low SD with poor coverage means the
  intervals are too tight (overconfident).

**Good result:** Samplers with similar post_std per element. Large differences suggest one
sampler is systematically more or less uncertain than the others about this quantity.

---

### delta/rmse/plots/ - Absolute error of Delta elements

**Files:** `delta_rmse_elements_c2_kt{k}.png` (one per k_true)

**What is shown:** For each Delta element, a boxplot per sampler of `|post_mean - true_value|`.
Unlike the bias plot, the sign is dropped, so positive and negative errors do not cancel.
Each point is one seed.

**Y-axis:** Free scale per panel (always >= 0).

**How to read it:**
- Low values - the posterior mean is close to the truth regardless of direction.
- A sampler can have near-zero bias (errors cancel across seeds) but high absolute error
  (large errors in both directions). Reading bias and absolute error together distinguishes
  systematic error from high variance.

**Good result:** Small absolute errors, similar across samplers. A sampler with larger absolute
error but smaller bias is noisy but unbiased; a sampler with larger bias is systematically off.

---

### delta/coverage/plots/ - 95% CI coverage of Delta elements

**Files:** `delta_coverage_c2_kt{k}.png` (one per k_true), `delta_coverage_by_ktrue_c2.png`

**What is shown:** The empirical 95% credible interval coverage rate for each Delta element.
Coverage = (number of seeds where the true value falls inside the posterior 95% CI) / 100.
Displayed as a percentage.

**Chart type:** Bar chart. One bar per sampler per element. Y-axis zoomed to 80-100%.

**Reference line:** Dashed line at 95% - the nominal target.

**How to read it:**
- Bar at 95% - the credible interval is well-calibrated: 95 out of 100 datasets have the true
  value inside the interval, as expected.
- Bar below 95% - undercoverage: the intervals are too narrow (overconfident).
- Bar above 95% - overcoverage: the intervals are too wide (conservative).
- `delta_coverage_by_ktrue_c2.png` shows all four k_true values side by side (dodged bars per
  sampler), making it easy to see whether coverage degrades as the model becomes more
  overspecified (k_model=5 fitted to fewer true components).

**Good result:** All bars close to 95% across elements and k_true values. Coverage well below
90% for any element signals a reliability problem with that sampler's uncertainty quantification.

## Beta - individual-level utilities

Beta_i is the vector of utility weights for decision-making unit i. With 330 units and 4
parameters, there are 330 x 4 = 1320 individual betas per run. Because comparing 1320 values
directly is impractical, each run is summarized to a single scalar per parameter by aggregating
over the 330 units. Each point in the plots below represents one replicate seed (dataset).

Parameters: Alt1, Alt2, Alt3 (alternative-specific intercepts), Price (price sensitivity).

### beta/bias/plots/ - Bias of individual betas

**Files:** `beta_bias_c2_kt{k}.png` (one per k_true)

**What is shown:** For each of the 4 parameters, a boxplot per sampler of the mean signed
error `mean(post_mean_i - true_i)` averaged over all 330 decision units. Each point is one
seed. Y-axis is free per panel.

**How to read it:** Same logic as the Delta bias plot. A box centered at zero means the sampler
does not systematically over- or underestimate individual utilities for that parameter. Spread
across seeds reflects how much recovery quality varies with the dataset.

**Good result:** Boxes centered near zero for all parameters and samplers. Systematic bias in
one sampler but not others indicates a problem with that sampler's individual recovery.

---

### beta/rmse/plots/ - RMSE of individual betas

**Files:** `beta_rmse_c2_kt{k}.png` (one per k_true)

**What is shown:** For each parameter, a boxplot per sampler of
`RMSE = sqrt( mean( (post_mean_i - true_i)^2 ) )` computed across 330 units. Each point is
one seed (one RMSE value that summarizes 330 individual errors for that parameter on that
dataset). Y-axis is free per panel.

**How to read it:**
- Each point answers: on this dataset, how far were the 330 posterior means from the 330 true
  betas for this parameter, on average?
- The box summarizes how this typical error varies across 100 datasets.
- A higher box means the sampler struggles more to recover individual-level heterogeneity for
  that parameter.
- Free y-axis per panel: a large RMSE in Alt1 does not compress the scale for Price.

**Good result:** Low median RMSE, similar across samplers. RMSE is always >= 0, so smaller is
better. Differences in median RMSE between samplers indicate that one sampler tracks individual
preferences more accurately.

---

### beta/coverage/plots/ - 95% CI coverage of individual betas

**Files:** `beta_coverage_c2_kt{k}.png` (one per k_true), `beta_coverage_by_ktrue_c2.png`

**What is shown:** The fraction of the 330 individual 95% credible intervals that contain the
true beta_i, averaged across seeds, expressed as a percentage. One bar per sampler per
parameter. Y-axis zoomed to 80-100%; dashed line at 95%.

**How to read it:** Same interpretation as Delta coverage but at the individual level. A bar
near 95% means the sampler's individual posteriors are well-calibrated: for a given decision
unit, its true beta_i is inside the posterior 95% CI in about 95 out of 100 datasets.

- `beta_coverage_by_ktrue_c2.png` shows all four k_true values side by side with dodged bars,
  so you can read off whether individual-level calibration is maintained as the mixture model
  becomes more overspecified.

**Good result:** All bars near 95%. Undercoverage at the individual level means the sampler
produces overconfident estimates of individual preferences - relevant if the model is used for
individual-level predictions or targeting.

## Runtime

### runtime/plots/ - Sampling time

**Files:**
- `runtime_nuts_c2_by_ktrue.png` - NUTS only, dual y-axis (minutes left / hours right)
- `runtime_hmc_c2_by_ktrue.png` - HMC only, minutes
- `runtime_bayesm_c2_by_ktrue.png` - bayesm only, minutes
- `runtime_samplers_c2_by_ktrue.png` - all three samplers on one log-scale plot

**What is shown:** Wall-clock time of the sampler call (excluding data loading and output
writing) in minutes (or hours for NUTS). Each boxplot pools 100 seeds per k_true level.

**How to read it:**
- Per-sampler plots: how does runtime scale with k_true for that sampler? A flat profile means
  the sampler's cost is insensitive to the number of true components; an upward slope means
  overspecification (k_model=5 > k_true) makes the sampler slower.
- Combined log-scale plot: compares all three on the same axis. Log scale is used because NUTS
  runtimes on overspecified problems can be an order of magnitude larger than HMC or bayesm.

**Key finding context:** NUTS trajectory cost grows substantially on overspecified mixtures
(more leapfrog steps per sample as the posterior becomes more complex), while HMC with a fixed
step budget and bayesm's Gibbs steps remain approximately flat across k_true values. This is
visible as a rising NUTS box with increasing k_true on the combined plot.

**Good result (from a practitioner perspective):** Faster runtime for equivalent or better
recovery. HMC and bayesm achieve roughly similar recovery quality to NUTS at a fraction of the
wall-clock cost on overspecified problems.
